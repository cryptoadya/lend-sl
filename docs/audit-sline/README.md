## Техническое обоснование ключевых замечаний

### Мобильное меню

В исходной версии класс `body.nav-open` запрещал прокрутку страницы, а фиксированное меню не имело ограничения по высоте и собственной прокрутки. В `aed915b` для открытого меню добавлены `max-height` и `overflow-y: auto`: [global.css](../../src/styles/global.css#L204). Смена desktop breakpoint теперь обрабатывается через `matchMedia`: [script.js](../../public/script.js#L38).

### Контактная форма

В `a234536` форма получила явные `method="post"` и `action="/api/contact"`, а `novalidate` удалён: [ContactCTA.astro](../../src/components/ContactCTA.astro#L29). При работающем JavaScript сохраняется прежняя JSON-отправка с ответом внутри страницы. Без JavaScript браузер отправляет `application/x-www-form-urlencoded` в тот же API, после чего получает `303` на страницу успеха или ошибки без персональных данных в URL. Обе страницы имеют `noindex` и исключены из sitemap.

API разбирает JSON и URL-encoded форму в одну структуру, после чего применяет общую серверную валидацию, honeypot и тот же Resend-путь: [contact.ts](../../functions/api/contact.ts#L138). Автотест с подменённым Resend подтверждает, что native POST формирует такое же письмо; опубликованный `dev` проверен только в безопасном `CONTACT_MODE=mock`.

В `3cced12` клиентская логика переведена на общий `checkValidity()`/`reportValidity()`, поэтому нативный `type="email"` блокирует неверный формат до запроса. Custom validity имени, сообщения и согласия очищается на `input`/`change`: [script.js](../../public/script.js#L60).

API выполняет повторную серверную проверку, что правильно. В `1c89a75` разбор запроса дополнительно укреплён: JSON принимается только как объект, тело читается потоково максимум до 10 000 байт независимо от `Content-Length`, а внешний запрос к Resend отменяется через 8 секунд: [contact.ts](../../functions/api/contact.ts#L193). Сетевые ошибки и timeout возвращают контролируемый ответ без вывода персональных данных в лог.

Rate limit/Turnstile поверх honeypot решено пока не добавлять: фактического спама нет. Вопрос остаётся на наблюдении и возвращается в работу, если появятся автоматические заявки или заметная нагрузка.

### Переход к форме

В `123d1ed` форма получила отдельный якорь `#kontaktformular`, а главный CTA ведёт прямо к нему: [Hero.astro](../../src/components/Hero.astro) и [ContactCTA.astro](../../src/components/ContactCTA.astro). Для якоря используется тот же `scroll-margin-top: 92px`, что и для разделов, поэтому sticky header не перекрывает первое поле.

До исправления на экране `390 × 844` верх формы находился на `885px` и не был виден. После исправления локальная версия и Cloudflare Preview показывают верх формы на `92px`; на desktop `1280 × 900` вся форма помещается до `864px`. Навигационная ссылка `Kontakt` намеренно продолжает вести к `#kontakt`, чтобы пользователь мог увидеть заголовок, телефон, e-mail и адрес.

Поскольку на desktop контакты и форма находятся в одной строке, в `c4baf61` для `#kontakt` добавлен desktop-only отступ `24px`: внутренний padding секции `68px` вместе с ним даёт те же `92px` до содержимого, что и якорь формы. На Preview оба перехода теперь дают `formTop=92px` и `sectionTop=24px`. На mobile правило не применяется: `Kontakt` открывает начало раздела, а CTA — форму.

### Доступность

В `dedc1dd` контактные поля получили отдельную границу `#87958d`: её расчётный контраст с белым фоном составляет `3,13:1`. Для `:focus-visible` используется тёмно-зелёный контур `#285d2c` толщиной `3px` с контрастом около `7,8:1`, а мягкая тень помогает заметить активное поле и при работе мышью: [global.css](../../src/styles/global.css#L804).

На локальной версии и Cloudflare Preview клавиша Tab переводит фокус с имени на телефон; контур остаётся видимым на desktop и mobile, горизонтального переполнения нет. Это улучшение не является сертификатом полного соответствия WCAG. Skip-link пока не добавляется по решению владельца: для текущей короткой одностраничной структуры он имеет меньший практический приоритет.

### Страница 404

В `c323b66` добавлена [404.astro](../../src/pages/404.astro), которая собирается в верхнеуровневый `404.html`. Страница объясняет ошибку, предлагает вернуться на главную или перейти к контактам и имеет `noindex, nofollow`. На Cloudflare Preview отдельно проверены несуществующий URL и отсутствующий asset-файл: оба ответа имеют статус `HTTP 404`. Вёрстка и переход на главную проверены в Playwright на desktop и mobile.

### SEO и производительность

Canonical URL, sitemap, robots, Open Graph и настоящая `404` уже присутствуют. Для завершения стоит добавить favicon, размеры/alt для social image, размеры крупных контентных изображений и длительный immutable-cache для хешированных файлов `/_astro/*`.
