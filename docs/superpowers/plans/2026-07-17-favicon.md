# S-Line Favicon Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Create and connect a small-size-optimized S-Line favicon based on the existing house-and-heart brand mark.

**Architecture:** Keep one deterministic SVG master in `public/` and commit two PNG derivatives generated from that master. Connect all three assets through the shared Astro layout so every generated page receives identical favicon markup.

**Tech Stack:** Astro 7, SVG, macOS Quick Look and `sips` for PNG rasterization, Node.js test runner.

## Global Constraints

- Use `#EAF4E7` for the rounded background and `#285D2C` for the house and heart.
- The SVG master uses `viewBox="0 0 512 512"`.
- Do not add text, gradients, shadows, a web app manifest, or changes to the header logo.
- The heart must remain a solid shape and the simplified house must omit the small door detail.
- Produce `public/favicon.svg`, `public/favicon-32x32.png`, and `public/apple-touch-icon.png`.

- [x] **Step 1: Write the failing favicon integration test**

Append this test to `tests/content.test.mjs`:

```js
test("every page receives the S-Line favicon assets from the shared layout", async () => {
  const [layout, svg, png32, appleTouch] = await Promise.all([
    readProjectFile("src/layouts/BaseLayout.astro"),
    readProjectFile("public/favicon.svg"),
    readFile(new URL("../public/favicon-32x32.png", import.meta.url)),
    readFile(new URL("../public/apple-touch-icon.png", import.meta.url)),
  ]);

  assert.match(
    layout,
    /<link rel="icon" href="\/favicon\.svg" type="image\/svg\+xml"\s*\/>/,
  );
  assert.match(
    layout,
    /<link rel="icon" href="\/favicon-32x32\.png" type="image\/png" sizes="32x32"\s*\/>/,
  );
  assert.match(
    layout,
    /<link rel="apple-touch-icon" href="\/apple-touch-icon\.png" sizes="180x180"\s*\/>/,
  );
  assert.match(svg, /viewBox="0 0 512 512"/);
  assert.match(svg, /#EAF4E7/);
  assert.match(svg, /#285D2C/);
  assert.ok(png32.byteLength > 0);
  assert.ok(appleTouch.byteLength > 0);
});
```

- [x] **Step 4: Rasterize the SVG into the two required PNG sizes**

Run:

```bash
FAVICON_TMP_DIR="$(mktemp -d /private/tmp/sline-favicon.XXXXXX)"
qlmanage -t -s 512 -o "$FAVICON_TMP_DIR" public/favicon.svg
sips -z 32 32 "$FAVICON_TMP_DIR/favicon.svg.png" --out public/favicon-32x32.png
sips -z 180 180 "$FAVICON_TMP_DIR/favicon.svg.png" --out public/apple-touch-icon.png
```

Expected: `sips` reports both output paths and creates exact 32 × 32 and 180 × 180 PNG files.
